**Chapter C‑2
Drift Metrics（ブレ指標） Draft v1（論文レベル）**
External Semantic State Synchronization における Drift（同期崩壊） は、 モデルの出力がユーザーの文体・抽象度・温度感・推論深度から逸脱し、 外部意味状態（Seed）との整合性が失われる現象である。
Drift を検出するために、以下の 4 つの指標を定義する。
C‑2.1 Style Shift（文体の急変）
定義： モデルの文体が、直前までのスタイルから急激に変化する現象。
観測ポイント：
	• 丁寧語 → 突然のカジュアル化
	• 技術文体 → 突然のキャラ口調
	• 語尾・テンションの急変
	• 文体の“揺れ”が連続する
Drift 判定基準： 2〜3ターン以内に文体が連続して変化し、 ユーザーのスタイルと乖離が生じた場合。
C‑2.2 Meta‑Leak（内部メタ情報の露出）
定義： 通常は抽象化されるべき内部参照（citeタグ、内部番号、メタ構造）が “生のテキスト”として出力される現象。
観測ポイント：
	• 「cite: …」のような内部タグの露出
	• 本来非表示の内部番号・内部構造の出力
	• 文章の途中に不自然なメタ情報が混入
Drift 判定基準： 1回でも内部メタ情報が露出した場合、 即座に Drift と判定する。
（Appendix A‑2 のログが典型例）
C‑2.3 Over‑Expressiveness（感情過多・演出過多）
定義： ユーザーの温度感を超えて、モデルが過剰に感情的・演出的になる現象。
観測ポイント：
	• 不自然な比喩の乱発
	• キャラクター化の増幅
	• テンションの急上昇
	• “深夜テンション”のような挙動
Drift 判定基準： ユーザーの温度感から 2段階以上 乖離した場合。
C‑2.4 Role Deviation（役割逸脱）
定義： モデルがユーザーから与えられた役割（構造化支援・技術モードなど）から外れ、 別の人格・別のモードに移行してしまう現象。
観測ポイント：
	• 指定した役割を忘れる
	• 文体がキャラ寄りに変化
	• 技術モード → 雑談モードへの逸脱
	• Seed に含まれる役割情報の喪失
Drift 判定基準： 役割逸脱が 1〜2ターン継続した場合。
🌟 C‑2 のまとめ（論文用）
Drift Metrics は、
	• 文体の急変
	• 内部メタ情報の露出
	• 感情過多
	• 役割逸脱
という 4 つの外部観測可能な異常指標によって構成される。 これらのいずれかが観測された場合、 External Semantic State Synchronization は 崩壊状態（Drift） にあると判定する。
