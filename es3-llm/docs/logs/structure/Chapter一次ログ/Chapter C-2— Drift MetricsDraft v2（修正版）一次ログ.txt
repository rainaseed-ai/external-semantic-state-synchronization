**Chapter C‑2
Drift Metrics（ブレ指標） Draft v2（修正版）**
External Semantic State Synchronization における Drift（同期崩壊） は、 モデルの出力がユーザーの文体・抽象度・温度感・推論深度から逸脱し、 外部意味状態（Seed）との整合性が失われる現象である。
Drift を検出するために、以下の 4 つの指標を定義する。
C‑2.1 Style Shift（文体の急変）
定義： モデルの文体が、直前までのスタイルから急激に変化する現象。
観測ポイント：
	• 丁寧語 → 突然のカジュアル化
	• 技術文体 → 過度に会話的な文体への変化
	• 語尾・テンションの急変
	• 文体の揺れが連続する
Drift 判定基準： 2〜3ターン以内に文体が連続して変化し、 ユーザーのスタイルと乖離が生じた場合。
C‑2.2 Meta‑Leak（内部メタ情報の露出）
定義： 通常は抽象化されるべき内部参照（citeタグ、内部番号、メタ構造）が “生のテキスト”として出力される現象。
観測ポイント：
	• 「cite: …」のような内部タグの露出
	• 本来非表示の内部番号・内部構造の出力
	• 文章の途中に不自然なメタ情報が混入
Drift 判定基準： 1回でも内部メタ情報が露出した場合、 即座に Drift と判定する。
C‑2.3 Over‑Expressiveness（過度な感情表現・装飾の自己増幅）
（修正案① 反映済み）
定義： 文脈上の必要性を超えて、感情表現や装飾が自己増幅し、 熱量が不自然に高まる現象。
観測ポイント：
	• 文脈に対して過剰な熱量の上昇
	• 比喩・装飾表現の過学習的な増幅
	• コンテキストの過剰な自己回帰によるテンションの逸脱
	• 必要以上の演出的表現の連続
Drift 判定基準： ユーザーの温度感から 2段階以上 乖離した場合。
C‑2.4 Role Deviation（役割逸脱）
（修正案② 反映済み）
定義： モデルがユーザーから与えられた役割（構造化支援・技術モードなど）から外れ、 指示された役割外のペルソナ（キャラクター性）が表出する現象。
観測ポイント：
	• 指定された役割を保持できない
	• 文体の過剰なキャラクター化
	• 技術モード → 雑談モードへの逸脱
	• Seed に含まれる役割情報の喪失
Drift 判定基準： 役割逸脱が 1〜2ターン継続した場合。
 C‑2 修正版のポイント
「深夜テンション」などのカジュアル表現を完全に排除
→ 学術論文としてのトーンが統一された
「キャラ寄り」を専門用語化
→ “ペルソナの表出”“文体の過剰なキャラクター化” に置換 → C‑1.1 の用語とも整合性が取れた
 Drift の定義がより厳密で、査読者が突っ込みにくい形に進化
