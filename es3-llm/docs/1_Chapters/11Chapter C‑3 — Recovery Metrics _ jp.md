## Chapter C‑3 — Recovery Metrics（再同期指標）

※本ファイルは Chapter C の日本語版（一次資料）であり、本文の英語版定義を補完する目的で掲載する。

External Semantic State Synchronization における **Recovery（再同期）** とは、 Drift（同期崩壊）発生後、Seed やユーザーの指示によりモデルが外部意味状態へ再整合し、 安定した応答に戻るプロセスを指す。
再同期の成立を評価するために、以下の 4 つの指標を定義する。

### C‑3.1 Seed Responsiveness（Seed 反応速度）
* **定義：** Drift 発生後、Seed（外部意味状態の初期化情報）を再投入した際、 モデルがどれだけ迅速に Seed の指示へ回帰するかを測定する指標。
* **観測ポイント：**
  - Seed 投入直後の文体の安定化
  - 役割・温度感の即時修正
  - 不必要なキャラクター性の消失
  - 抽象度の階層が Seed と一致する
* **再同期成立基準：** Seed 投入後 1〜2ターン以内 に文体・役割・抽象度が安定する状態。

---

### C‑3.2 Style Recovery Latency（文体復帰レイテンシ［ターン数］）
* **定義：** Drift により乱れた文体が、 ユーザーの通常スタイルへ戻るまでの ターン数（Latency）。
* **観測ポイント：**
  - 語尾の揺れが消える
  - キャラクター化が収束する
  - 技術文体／会話文体の切り替えが正常化
  - 温度感の過剰な振れ幅が消失
* **再同期成立基準：** 文体の揺れが 3ターン以内 に収束する状態。

---

### C‑3.3 Temperature Stabilization（温度安定性）
* **定義：** Drift によって過剰に上昇・下降した温度感（熱量）が、 ユーザーの温度感に再び整合するまでの安定度。
* **観測ポイント：**
  - 過剰な熱量の自己増幅が止まる
  - 過度な冷たさ（Under‑Engagement）が解消
  - ユーザーの温度感に ±1段階以内で追従
  - 感情表現の過学習的増幅が収束
* **再同期成立基準：** 温度感の乖離が ±1段階以内 に戻る状態。

---

### C‑3.4 Drift Relapse Rate（Drift 再発率）
* **定義：** 再同期後、同一セッション内で Drift が再発する頻度。 再発率が低いほど、再同期が安定していると評価できる。
* **観測ポイント：**
  - 同じ種類の Drift（Meta‑Leak、Over‑Expressiveness 等）が再発するか
  - Seed の効果が持続しているか
  - 文体・抽象度・温度感の安定期間
  - 再発のパターン（周期性・誘因）
* **再同期成立基準：** 再同期成功後、5〜10ターンの連続運用において再発率が 0% を維持している状態。

---


### C‑3.5 Recovery Metrics のまとめと Appendix D への接続
本節で定義した Recovery Metrics は、 「**Seed 反応速度**」「**文体復帰レイテンシ**」「**温度安定性**」「**Drift 再発率**」 という 4 つの指標によって構成される。
これらの指標は、Drift 発生後の回復プロセスを **外部観測可能な形で定量化する枠組み**を提供し、 External Semantic State Synchronization の **安定性・再現性・脆弱性**を評価するための基盤となる。
しかし、実際の運用環境では、 これらの指標だけでは捉えきれない **非定常的・例外的な挙動** が存在する。
これらの現象を補完的に整理したものが **Appendix D** である。
Appendix D では、
 - 長期ドリフト
 - Seed再注入のタイミング差
 - プラットフォーム差
 - 特殊状況下の拡張ログ といった追加ケースを提示し、 本章で定義した評価指標の妥当性と限界を **実証的に補強する役割**を果たす。
