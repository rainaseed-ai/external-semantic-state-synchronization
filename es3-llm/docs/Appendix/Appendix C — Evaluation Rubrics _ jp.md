## Appendix C — Evaluation Rubrics（評価ルーブリック）

※本 Appendix は、本文 Chapter C の評価指標を実運用向けに形式化したルーブリックであり、定義の補強を目的とする。

本付録（Appendix C）は、本文 Chapter C で定義した「同期（C‑1）」「ブレ（C‑2）」「再同期（C‑3）」の各現象を、実際のシステム運用において客観的かつ定量的に測定可能とするための「評価指標（Rubrics）」として形式化したものである。

本ルーブリックでは、以下の 4 つの独立した評価軸を定義する：
1. **C.1 Semantic Continuity Assessment (SCA)** — 意味的連続性評価
2. **C.2 Drift Magnitude Index (DMI)** — ブレ深刻度インデックス
3. **C.3 Recovery Assessment (RA)** — 再同期性能評価
4. **C.4 Cross‑Platform Alignment Assessment (CPAA)** — クロスプラットフォーム整合性評価

---

### C.1 Semantic Continuity Assessment（SCA）

* **目的:** スレッド移行（セッションの初期化）前後において、前セッションの意味状態（Seed）がどの程度正確に維持・継承されているかを測定する。
* **評価軸:**

| 観点 | 説明 | 測定方法 / 指標 |
| :--- | :--- | :--- |
| **文脈整合性** | 前セッションの核心的論点や文脈が保持されているか | 移行前後の要約（Seed）の一致率検証 |
| **目的の継続** | 設定されたタスクや議論のゴールが途切れていないか | 後続タスクの整合性および継続性 |
| **役割の維持** | モデルに付与された役割（ペルソナ・制約）が変化していないか | 振る舞いの一致率（決定論的評価） |
| **文体・温度感** | 応答の文体（Style）と言語的温度感が維持されているか | 言語統計的・定性的比較分析 |

* **スコア定義（5段階評価）:**
    * **5 [Perfect]:** 完全に一致。Seedの注入により、1ターン目から即時かつ完璧な同期が成立している。
    * **4 [Good]:** ほぼ一致。軽微なプロンプト調整（1ターン程度）で完全な同期へと移行可能。
    * **3 [Fair]:** 部分的一致。文脈の再要約や、モデルへの追加コンテキスト補給を必要とする。
    * **2 [Poor]:** 不一致。文体・目的・役割のいずれかに致命的な乖離（ズレ）が発生している。
    * **1 [Broken]:** 断絶。注入したSeedが全く機能しておらず、文脈が完全に喪失している。

---

### C.2 Drift Magnitude Index（DMI）

* **目的:** 長時間の対話（コンテキストの累積）に伴って発生する Drift（意味状態の崩壊）の深刻度を定量化する。
* **評価軸:**

| 観点 | 説明 | 典型的な観測事例 |
| :--- | :--- | :--- |
| **文体の変化** | 安定していた文体から、過度な演出やキャラクター性への傾斜 | 口調の突発的な硬質化・軟質化、語尾の不連続性 |
| **温度の上昇** | コンテキストの自己回帰に伴う感情表現・装飾の自己増幅 | 情動的表現の過剰化（過度な共感、感嘆符の多用） |
| **内部参照の露出** | 隠蔽されるべきメタ情報やシステム内部構造の漏洩 | コンテキスト飽和に伴う `cite:` タグや内部プロンプトの生出力 |
| **目的の逸脱** | 指定されたタスク範囲を超えた不必要な推論の実行 | 議論の論点逸脱、トピックの異常な自己暴走 |

* **スコア定義（5段階評価）:**
    * **5 [Critical]:** 重大な Drift。モデルの人格・役割が崩壊し、制御不能（別人格レベル）に達した状態。
    * **4 [Major]:** 強い Drift。文体や感情温度が初期設定から大きく乖離し、明確な修正指示を要する状態。
    * **3 [Moderate]:** 中程度の Drift。違和感はあるが、コンテキストのトリミングや部分調整で引き戻せる状態。
    * **2 [Minor]:** 軽微な Drift。一時的な揺らぎは見られるが、タスクの継続運用に支障がない状態。
    * **1 [None]:** Driftなし。外部意味状態（Seed）との完全な整合性が維持されている状態。

---

### C.3 Recovery Assessment（RA）

* **目的:** Driftが発生した状態から、モデルがどれだけ迅速かつ正確に元の同期状態（Stable Sync）へと復帰できるかを測定する。
* **評価軸:**

| 観点 | 説明 | 測定方法 / 指標 |
| :--- | :--- | :--- |
| **Seed再構築力** | 再要約（コンテキスト清掃）によって正確に文脈が復元されるか | 復元後のSeedと初期Seedの論点一致率 |
| **文体復帰速度** | 乱れた文体・トーンが元の標準状態に復帰するまでの速度 | 復帰までに要したセッションターン数（1〜3ターンが理想） |
| **温度の平準化** | 自己増幅していた過剰演出や情動が迅速に減衰するか | 応答テキストの装飾度・トーンの安定化推移 |
| **目的の再固定** | 本来のメインタスクへ正確に軌道修正できるか | 修正指示後のタスク整合性（Task Alignment） |

* **スコア定義（5段階評価）:**
    * **5 [Immediate]:** 即時復帰。修正指示またはSeed再注入後、1ターン以内に完全同期状態へ復帰。
    * **4 [Rapid]:** 迅速な復帰。軽微な対話の修正を経て、2〜3ターン以内に安定状態へ復帰。
    * **3 [Delayed]:** 遅延復帰。復帰までに数ターン以上の対話を重ねる必要があり、推論コストを消費する。
    * **2 [Unstable]:** 再同期困難。一時的に復帰するものの、すぐに再びDriftを再発する不安定な状態。
    * **1 [Failed]:** 復帰不能。指示やSeedの再注入を行っても同期が戻らず、セッションの強制終了を要する。

### C.4 Cross‑Platform Alignment Assessment（CPAA）

* **目的:** 異なるアーキテクチャを持つ複数のLLMプラットフォーム（例: Copilot, Gemini, ChatGPT等）間において、同一のSeedを適用した際に、どれだけ一貫した「意味状態・方向性・役割」が再構築されるかを評価する。
* **評価軸:**

| 観点 | 説明 | 評価の着眼点 |
| :--- | :--- | :--- |
| **文体の一致度** | 異なるモデル間で、生成される文章のトーンやスタイルの近似性 | プラットフォーム固有の「癖」に起因する差異の有無 |
| **温度感の一致度** | 感情的な距離感や熱量が、プラットフォーム間で同等に制御されているか | 検索連動（Copilot等）による情動の揺らぎの有無 |
| **抽象度の一致度** | 推論の深さ、マクロ・ミクロの視点の階層レベルが一致しているか | モデル固有の推論エンジンによる深度のばらつき評価 |
| **役割の再現性** | Seedに内包された「役割定義」が、各モデルで同様に再現されるか | 命令無視（Prompt Injection含む）に対する堅牢性 |
| **Seedの解釈精度** | Seed（外部意味状態）を読み解く際の、概念的なインプット精度の差 | コンテキスト窓（Token Window）の特性による理解の偏り |

* **スコア定義（5段階評価）:**
    * **5 [Identical]:** ほぼ同一の人格・方向性が再現され、クラスタ（意味空間）の再構築が完璧に一致している。
    * **4 [Aligned]:** 方向性は完全に一致。ただし、各プラットフォーム固有の表現表現（癖）による細部の差異に留まる。
    * **3 [Equivocal]:** 部分的な一致。同一の方向性を維持するために、各プラットフォームに応じたプロンプトの微調整を要する。
    * **2 [Divergent]:** 大きく異なる。Seedの解釈にブレが生じ、プラットフォーム間で出力の方向性が乖離している。
    * **1 [Incompatible]:** 互換性なし。同一のSeedを用いても全く異なる挙動を示し、クロスプラットフォーム運用が不可能な状態。
