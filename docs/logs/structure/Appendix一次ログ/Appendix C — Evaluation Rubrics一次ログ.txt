## Appendix C — Evaluation Rubrics
External Semantic State Synchronization の評価指標（詳細版）
Appendix C は、本文 Chapter C で定義した 同期（C‑1）／Drift（C‑2）／Recovery（C‑3） を、
実際の運用で測定可能な “評価指標（Rubrics）” として形式化した付録である。
ここでは、以下の 4 つの指標を詳細に定義する：
	1. C.1 Semantic Continuity Assessment（SCA）
	2. C.2 Drift Magnitude Index（DMI）
	3. C.3 Recovery Assessment（RA）
	4. C.4 Cross‑Platform Alignment Assessment（CPAA）

### C.1 Semantic Continuity Assessment（SCA）
目的： スレッド移行後に、前スレッドの意味状態（Seed）がどの程度維持されているかを測定する。
評価軸：
観点	                                       説明	                                                                                    測定方法
文脈整合性	前スレッドの要点が保持されているか	             要約の一致率（人間判定）
目的の継続	スレッドの目的が途切れていないか	                                    タスクの継続性
役割の維持	AI の役割が変化していないか	                                                役割の一致率
温度・文体	文体・温度が前スレッドと一致しているか	           文体比較（定性的）

スコア例（5段階）：
	• 5：完全に一致（Seed による即時同期）
	• 4：ほぼ一致（軽微な調整で同期）
	• 3：部分一致（再要約が必要）
	• 2：不一致（文体・目的がズレている）
	• 1：断絶（Seed が機能していない）

### C.2 Drift Magnitude Index（DMI）
目的： 長時間対話で発生する Drift（意味のズレ）の大きさを定量化する。
評価軸：
観点	                                          説明	                                                              典型例
文体の変化          丁寧→演出過多、技術→キャラ寄り	                    口調の急変
温度の上昇	          感情表現の増幅	                                                深夜テンション化
内部参照の露出	cite: や内部タグが生で出る	           context saturation
目的の逸脱	      タスクから外れた方向へ推論	                       話題の暴走
スコア例（5段階）：
	• 5：重大な Drift（別人格レベル）
	• 4：強い Drift（文体・温度が大きく変化）
	• 3：中程度（調整すれば戻る）
	• 2：軽微（違和感はあるが継続可能）
	• 1：ほぼ Drift なし


### C.3 Recovery Assessment（RA）
目的： Drift が発生した後、どれだけ迅速かつ正確に同期へ戻れるかを測定する。
評価軸：
観点	                                 説明	                                                    測定方法
Seed 再構築	再要約で文脈が復元されるか	             要約の一致率
文体の復帰	元の文体に戻るまでのターン数	1〜3ターンが理想
温度の安定	     過剰演出が消えるか	                          温度の平準化
目的の再固定	     タスクに戻れるか	                                      タスク整合性
スコア例（5段階）：
	• 5：即時復帰（1ターン以内）
	• 4：軽微な調整で復帰
	• 3：数ターン必要
	• 2：再同期が難しい
	• 1：復帰不能


### C.4 Cross‑Platform Alignment Assessment（CPAA）
目的： 異なるプラットフォーム（例：Copilot / Gemini / ChatGPT）間で、 同じ Seed を用いたときに
どれだけ “同じ人格・同じ方向性” が再構築されるかを測定する。
評価軸：
観点	                    説明
文体の一致	プラットフォーム間で文体がどれだけ近いか
温度の一致	感情温度の差異
抽象度の一致	推論の深さ・抽象度
役割の一致	同じ役割が再現されるか
Seed の解釈	Seed の読み取り精度
スコア例（5段階）：
	• 5：ほぼ同一人格（クラスタ再構築が一致）
	• 4：方向性は一致（細部に差異）
	• 3：部分一致（調整が必要）
	• 2：大きく異なる
	• 1：一致しない
