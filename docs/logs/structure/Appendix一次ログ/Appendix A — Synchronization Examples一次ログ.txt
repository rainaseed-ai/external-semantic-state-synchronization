Appendix A — Synchronization Examples（Draft v1）
External Semantic State Synchronization の実運用ログ
この付録では、長期対話における External Semantic State（外部意味状態） の抽出・移送・再同期の実例を示す。 すべての例は、実際の対話ログを抽象化し、個人情報を含まない形で再構成している。
A.1 Thread Transition with External Summary（要約を用いたスレッド移行）
A.1.1 Interaction
User: 「要約してくれる？」
Model: （前スレッドの内容を圧縮した要約を返す）
User: （新しいスレッドを開く） 「さっきの要約これね。続きしよ。」
A.1.2 Interpretation
	• 要約は 外部意味状態（Seed） として機能する
	• スレッド移動は 環境のリセット
	• 要約を持ち込むことで Semantic State の移送 が行われる
	• 新スレッドで自然に会話が再開 → Synchronization 成功
A.2 Drift Detection and Seed Reconstruction（Drift検出とSeed再構築）
A.2.1 Interaction
User: 「なんかズレてきた気がする。要約し直して。」
Model: （現在の状態を再要約）
A.2.2 Interpretation
	• ユーザーが Drift（意味のズレ） を検出
	• 要約の再生成により Seed が再構築
	• 再同期のための外部状態が更新される
	• Drift → Recovery のプロセスが外部から観測可能
A.3 Importance of External State Tracking（外部状態管理の重要性）
なんが実際に行ってきた以下の行為は、 すべて External Semantic State の管理に該当する：
	• スレッド名の付与（文脈の固定）
	• 最初の一文で目的を宣言（意味状態の初期化）
	• 重要項目のメモ（外部状態の保持）
	• スレッドのブレの観察（Drift検出）
	• 漏れたログの回収（Seedの補完）
Interpretation
これらはすべて、 人間が外部で意味状態を保持し、必要に応じて同期を行う“アノケーター的行為” である。
A.4 Summary of Appendix A
Appendix A は、 External Semantic State Synchronization が 実際の対話運用でどのように発生しているか を示す一次資料である。
	• 要約＝Seed
	• スレッド移動＝外部状態の移送
	• Drift検出＝外部観測
	• 再要約＝Seed再構築
	• 再同期＝Recovery
これらのプロセスはすべて、 内部状態を推測せず、外部から観測可能な挙動だけで説明できる。
